# DEMO_SCRIPT_20260422

## Recommended Duration
2 to 3 minutes

## Demo Structure

### 1. Problem Introduction
Say:
"This project is a Personal Spending Insight Dashboard designed for students. It helps users understand where their money goes, identify unusual spending, and reflect on non-essential consumption."

### 2. Show the Dashboard Entry
Open the deployed app or local app.
Say:
"The dashboard can load a built-in sample dataset or accept an uploaded CSV file."

### 3. Show KPI Cards
Point to:
- Total Spending
- Avg Transaction
- Outlier Count
- Health Score

Say:
"These KPIs summarize the current spending profile at a glance."

### 4. Show Category and Trend Charts
Point to:
- category bar chart
- monthly trend chart

Say:
"These charts explain where spending is concentrated and when spending peaks happened."

### 5. Show Outlier Transactions
Point to the outlier table.
Say:
"This section highlights transactions that are unusual relative to the normal range of their category."

### 6. Show Top Merchants and Analysis Report
Point to:
- Top Merchants
- Analysis Report

Say:
"The app does not stop at visualization. It also converts the results into an interpretable analysis report."

### 7. Close with Reflection
Say:
"The project focuses on explainable analytics rather than complex machine learning. It combines cleaning, mapping, feature engineering, outlier detection, scoring, and communication in a small Streamlit MVP."
